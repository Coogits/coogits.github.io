    Identifier

    An identifier is the name assigned to variables, functions, classes, and other entities in a program. It serves as a unique label that allows programmers to reference and manipulate data or functionality within a codebase.

    // Identifier example in C
int identifier = 0;
// the identifier's type is 'int' and it's value is 0

    Characteristics

    • Start without number:
      Many languages disallow identifiers starting with a digit (e.g., 1variable is invalid)

    • Limited special characters:
      Typically, only _ is allowed; other symbols (@, $, %) are restricted.

    • Case-sensitive:
      In most languages, variable and Variable are distinct (e.g., C, Java, Python).

    • Descriptive:
      Clear names improve code readability and maintainability.

    Namespace

    A namespace is a declarative region that provides a scope to group identifiers (such as variables, functions, classes, or modules) to prevent naming conflicts and organize code efficiently.

    Key concepts:

    • Encapsulation of Identifiers:
      Avoids name clashes by grouping related identifiers.
    • Hierarchical Organization:
      Supports nesting to structure large programs.
    • Language-Specific Implementations:
      Namespaces appear in different forms across languages.
    • Explicit Access Using Scope Operators:
      Access elements using :: (C++), . (Python, Java), etc.

    Scope

    Scope refers to the visibility and lifetime of an identifier (such as variables, functions, or objects) within a program. It defines where an identifier can be accessed or modified in the code.

    Key concepts:

    • Defines Accessibility:
      Determines where an identifier can be referenced.
    • Affects Lifetime:
      Controls how long an identifier exists in memory.
    • Varies by Language:
      Scope rules differ across programming languages.

    Examples

    Variable Identifier

    A variable identifier is the name assigned to a variable, representing a memory location storing a value.

    // Variable identifier
int varIdentifier = 0;
    • varIdentifier is the variable identifier for an integer (int).
    • Stores the value 0 in memory.
    • Can be modified later in the program:

    Constant Identifier

    A constant identifier refers to a named constant whose value cannot be changed after initialization.

    // constant(immutable) identifier
const int constIdentifier = 0;
// constant(preprocessor) identifier
#define PI 3.14
    • constIdentifier is the identifier for a constant integer.
    • The const keyword ensures the value remains unchangeable.
    • Trying to modify constIdentifier later in the code will result in a compilation error.

    Function Identifier

    A function identifier is the name assigned to a Function, used to define and call it. In the example:

    // function identifier
void fooIdentifier (void);
    • fooIdentifier is the function identifier.
    • It represents a function that takes no arguments (void) and returns nothing (void).
    • It is a declaration (not a definition), meaning the function's implementation is provided elsewhere.

    Struct Identifier

    A struct identifier names a structure type, allowing the creation of structured data with multiple members.

    // struct identifier
struct MyStruct { int member; };
    • MyStruct is a struct identifier.

    Typedef Identifier

    // Variable identifier
int varIdentifier = 0;
// constant identifier
const int constIdentifier = 0;
// function identifier
void fooIdentifier (void);
// struct identifier
struct MyStruct { int member; };
// typedef identifier
typedef unsigned int uint;
// enum identifier
enum Color { RED, GREEN, BLUE };
