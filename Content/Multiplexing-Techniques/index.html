    Multiplexing Techniques

    Multiplexing is a technique used in telecommunications to combine multiple signals or data streams into one transmission medium, allowing for the efficient utilization of the available bandwidth.
    The primary objective of multiplexing is to maximize the usage of a single communication channel or resource by enabling the transmission of several signals simultaneously without interference.
    Each signal is typically assigned its own time slot, frequency band, or phase, depending on the type of multiplexing used.

    Key Aspects

    • Combining Signals: Multiple signals (which could be data, voice, or video) are combined for transmission over a single physical medium (e.g., a fiber-optic cable, radio wave, or twisted pair).
    • Efficiency: The technique helps optimize the use of costly and limited resources such as bandwidth and physical medium (wires, frequencies, etc.).
    • Separation of Signals: Even though the signals are transmitted together, they are logically separated at the receiver end, often through specialized demultiplexing techniques.

    Access Types in Multiplexing

    Centralized Access

    • Definition: In centralized access, a central entity governs access to the shared communication medium. This entity manages the allocation of resources to avoid conflicts and ensure efficient transmission.
    • Examples:
      • Polling: A central controller sequentially requests each device for data transmission, allowing devices to send data only when polled.
      • Token Passing: A control token circulates among devices in a network. Only the device holding the token is permitted to transmit, ensuring orderly access.

    Distributed Access

    • Definition: In distributed access, multiple devices share the communication medium without a central controller. Access is managed by individual devices through various protocols that handle contention and resource allocation.
    Deterministic Access
    • Definition: In deterministic access schemes, transmission opportunities are predetermined based on a fixed schedule or predefined allocation, ensuring that no conflicts occur during transmission.
      • Examples:
        • Time Division Multiplexing (TDM): Devices are allocated specific time slots for transmission, ensuring that only one device transmits during each slot.
        • Frequency Division Multiplexing (FDM): Devices are assigned unique frequency bands within the spectrum, ensuring simultaneous transmissions without overlap.
    Code Division Multiple Access (CDMA)
    • Definition: CDMA is a spread-spectrum technique where each device is assigned a unique code for transmission. Multiple devices can transmit simultaneously within the same frequency band, with each signal being distinguishable due to its unique code.
      • Example:
        • Wideband CDMA (WCDMA): Commonly used in cellular networks, where each communication is encoded with a unique spreading code to allow multiple simultaneous transmissions.
    Contention-Based Access
    • Definition: In contention-based access, devices compete for the shared medium. Devices must detect channel availability and handle possible transmission conflicts, often through backoff strategies and retransmission mechanisms.
      • Examples:
        • Carrier Sense Multiple Access (CSMA): Devices listen to the medium to detect if it is idle before attempting transmission. If the medium is busy, the device defers transmission.
        • CSMA with Collision Detection (CSMA/CD): After transmission, devices monitor the channel for collisions and retransmit if necessary. This technique is commonly used in Ethernet networks.
        • CSMA with Collision Avoidance (CSMA/CA): Used primarily in wireless networks (such as Wi-Fi), where devices attempt to avoid collisions by deferring transmission until the channel is clear.

    Static vs. Dynamic Multiplexing

    Static Multiplexing

    • Definition: In static multiplexing, resources are divided into fixed segments and allocated to users or channels in advance, without any changes during transmission. The allocation of bandwidth or time slots is predetermined, and each user gets a dedicated portion of the available capacity.
    • Channel Types: Physical Channels. In static multiplexing, the division of these physical channels is predetermined, with each user or signal assigned a specific portion (e.g., time slots in TDM or frequency bands in FDM).
    • Examples:
      • Time Division Multiplexing (TDM): Divides time into fixed slots, with each user allocated a specific slot for transmission.
      • Frequency Division Multiplexing (FDM): Divides the available frequency spectrum into fixed bands, with each user assigned a specific frequency for communication.

    Dynamic Multiplexing

    • Definition: Dynamic multiplexing adjusts the allocation of resources in real-time based on demand or network conditions. It allows more flexible and efficient use of available bandwidth or time, adapting to changing traffic loads or user requirements.
    • Channel Types: Logical Channels. In dynamic multiplexing, the allocation of these logical channels is flexible and can adjust based on demand or real-time conditions (e.g., in on-demand access methods like CSMA/CA).
    Dynamic Multiplexing Types
    • On-Demand Allocation: Resources are allocated dynamically based on the immediate needs of users or devices. This method adjusts the allocation as traffic conditions change.
      • Examples:
        • Carrier Sense Multiple Access with Collision Avoidance (CSMA/CA): Used in Wi-Fi networks, where devices dynamically sense the channel before transmitting to avoid collisions.
        • Random Access Protocols: Systems like Ethernet use dynamic access methods, where devices send data when the channel is free, dynamically allocating bandwidth as needed.
    • Pre-Allocation: Resources are pre-allocated but can be adjusted dynamically based on real-time traffic or user needs, offering a more flexible approach than static multiplexing.
      • Examples:
        • Asynchronous Transfer Mode (ATM): Bandwidth is pre-allocated to each user but can be adjusted based on demand, allowing more efficient use of network resources.
        • Packet-Switched Networks: Resources are allocated to data packets dynamically, such as in the case of the Internet, where the path and bandwidth can change based on network congestion or other conditions.

    Types of Multiplexing Techniques

    Time Division Multiplexing (TDM)

    • How It Works:
      TDM allocates different time slots to each signal on a shared communication channel. Each signal is transmitted in a sequence, with each one being given a specific time window for transmission.
    • Access Control:
      The access control is based on the timing sequence, ensuring that no two devices transmit at the same time, usually Static Allocated.
    • Applications:
      Common in digital communication systems, such as in telephone networks, where voice calls are digitized and transmitted over time slots in a multiplexed stream.

    Frequency Division Multiplexing (FDM)

    • How It Works:
      FDM divides the available bandwidth into several non-overlapping frequency bands. Each signal is modulated onto a different frequency carrier and transmitted simultaneously on these separate frequency bands.
    • Access Control:
      The control here is based on frequency allocation, ensuring that no two devices use the same frequency band.
    • Applications:
      Used in analog radio and television broadcasting, satellite communication, and early telephone networks (e.g., analog phone systems).

    Wavelength Division Multiplexing (WDM)

    • How It Works:
      A form of FDM used in optical fiber communication, where different signals are transmitted at different wavelengths (colors) of light. Each signal is assigned a different wavelength of light to carry data through optical fibers.
    • Applications:
      WDM is widely used in high-speed internet and long-distance communications via optical fiber, offering very high bandwidth and capacity.

    Code Division Multiplexing (CDM)

    • How It Works:
      In CDM, each signal is assigned a unique code, and all signals are transmitted simultaneously over the same frequency band. The receiver uses the code to distinguish between signals and recover them.
    • Access Control:
      The control is based on assigning unique codes to users, which allows them to share the same channel.
    • Applications:
      Used in wireless systems such as CDMA (Code Division Multiple Access) cellular networks.

    Space Division Multiplexing (SDM)

    • How It Works:
      SDM utilizes multiple physical paths (such as fiber-optic cables or antennas) to transmit multiple signals simultaneously. This is often done using multiple antennas or fibers, each carrying a separate stream of data.
    • Applications:
      Used in modern fiber-optic communication systems, MIMO (Multiple Input Multiple Output) antenna systems, and spatially separated communication networks.

    Polling and Token-based Access

    • Access Method:
      In polling-based access, a central controller or "polling" entity requests permission from each device to transmit. Each device responds in turn, and the central controller determines the transmission schedule.
    • Token-based Access:
      In token passing, as explained earlier, a token is passed around the network, and only the device holding the token is allowed to transmit. This ensures deterministic access.
    • Access Control:
      Polling and token-based methods provide centralized access control, as a central entity or token ensures orderly access to the shared medium.
    • Example:
      Token Ring networks use token passing for access control. In polling systems, systems like Modbus use a master-slave structure to control access to the network.

    Demand Assignment (DA)

    • Access Method:
      In DA, devices request access to the channel when they need to transmit, and the central controller grants access based on availability. This system allows devices to access the medium dynamically, as needed.
    • Access Control:
      The access control is based on demand, with the central controller managing which device gets access to the communication medium at any given time.
    • Example:
      Dynamic Frequency Assignment (DFA) systems in satellite communication use this type of access.

    Carrier Sense Multiple Access with Collision Detection (CSMA/CD)

    • Access Method:
      CSMA/CD is a contention-based access method where devices "listen" to the channel before transmitting to ensure it is not already in use. If two devices transmit simultaneously, a collision occurs, and both devices must retransmit after a random delay.
    • Access Control:
      The access control here is based on sensing the carrier (the channel) to determine whether it is idle or in use. If the channel is busy, devices must wait until it is free.
    • Example:
      Ethernet networks (such as those using the IEEE 802.3 standard) use CSMA/CD to manage data transmission.
