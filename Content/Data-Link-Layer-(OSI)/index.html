    Datalink Layer

    The Data Link Layer (Layer 2) of the OSI (Open Systems Interconnection) model is responsible for the reliable transfer of data frames between two directly connected nodes over a physical medium. It ensures that data is delivered error-free, in sequence, and without duplication, functioning as an interface between the physical layer and the network layer.

    Core Functions

    Framing

    • The Data Link Layer breaks the raw bit stream from the Physical Layer into frames, which are structured units of data that include both the payload (actual data) and control information (such as header and trailer).
    • A frame typically includes:
      • Frame header: Contains control information, such as source and destination addresses (MAC addresses), frame type, and other protocol-specific information.
      • Data: The actual payload or user data.
      • Frame trailer: Contains error detection information like a Cyclic Redundancy Check (CRC) or checksum.
    • The framing process ensures that the data stream can be recognized, distinguished, and properly reassembled by the receiver.

    Error Handling

    • One of the primary roles of the Data Link Layer is to ensure that data frames are transmitted correctly. It detects errors caused by noise or interference in the physical medium.
    • Error Detection: Common techniques include Parity Bits, Checksums, and Cyclic Redundancy Check (CRC), where the sender calculates a checksum based on the data and appends it to the frame. The receiver recalculates the checksum upon receipt and compares it to the one sent. If they do not match, an error is assumed.
    • Error Correction: Some Data Link Layer protocols (such as HDLC) can request retransmission of erroneous frames, ensuring reliable data transfer in cases of corruption. Others might use Automatic Repeat reQuest (ARQ) protocols to achieve this.

    Flow Control:

    • Flow control ensures that the sending device does not overwhelm the receiving device with data, especially when the receiving device has limited processing or buffer capacity.
    • The Data Link Layer employs various flow control methods:
      • Stop-and-wait: The sender waits for an acknowledgment of each frame before sending the next.
      • Sliding window: Allows the sender to send multiple frames before needing acknowledgment, enhancing efficiency.

    Medium Access Control (MAC):

    • The Medium Access Control (MAC) sublayer of the Data Link Layer is responsible for controlling access to the shared communication medium. This is essential in environments where multiple devices share the same physical medium (e.g., a wired or wireless network).
    • The MAC sublayer uses different techniques to avoid collisions (when two devices attempt to transmit at the same time) and to ensure fair access:
      • CSMA/CD (Carrier Sense Multiple Access with Collision Detection): Used in Ethernet networks, where devices "listen" for a clear channel before transmitting and back off if a collision is detected.
      • TDMA (Time Division Multiple Access): Divides the time into slots and assigns each device a specific time window to transmit.
      • CDMA (Code Division Multiple Access): Assigns a unique code to each device to distinguish signals on the same frequency.

    Addressing

    • The Data Link Layer uses MAC addresses to uniquely identify devices on a local network. A MAC address is a hardware address assigned to the network interface card (NIC) of each device and serves as a unique identifier for devices on a Local Area Network (LAN).
    • The MAC address is used by the Data Link Layer to direct frames to the correct device on the network. Unlike IP addresses at the Network Layer, which are used for routing across networks, MAC addresses are used for direct communication between devices within the same network.

    Link Management

    • The Data Link Layer is also responsible for managing the link between devices, ensuring that the connection is properly established, maintained, and terminated. It ensures that devices synchronize their transmission and reception of frames, and it manages the flow of frames to ensure that frames are delivered in order.
    • For example, in PPP (Point-to-Point Protocol), the Data Link Layer handles link establishment, authentication, and configuration management.

    Data Link Layer Protocols

    • Ethernet: The most widely used protocol for LANs. Ethernet operates in the Data Link Layer and defines frame structure, MAC addressing, and collision handling (CSMA/CD).
    • PPP (Point-to-Point Protocol): Used to establish direct connections between two devices, such as over dial-up or serial links. PPP provides authentication, error detection, and compression.
    • HDLC (High-Level Data Link Control): A bit-oriented protocol used for synchronous communication, widely used in point-to-point WAN links.
    • Wi-Fi (IEEE 802.11): A protocol that defines wireless LAN communication, operating both at the Data Link Layer and Physical Layer.

    The Relationship with Other Layers

    • Physical Layer (Layer 1): The Data Link Layer builds on the Physical Layer by providing the necessary data structures (frames) and mechanisms for error correction and flow control to ensure reliable data transmission.
    • Network Layer (Layer 3): The Data Link Layer interacts with the Network Layer by providing logical addressing through MAC addresses and supporting the routing of data frames between different devices on a network.

    Example: Ethernet and Wi-Fi in Data Link Layer

    • Ethernet: In Ethernet, devices are connected via cables, and data is sent as frames with source and destination MAC addresses. It employs CSMA/CD to avoid collisions when multiple devices attempt to transmit simultaneously.
    • Wi-Fi (IEEE 802.11): In wireless LANs, Wi-Fi provides similar functionality but over the air, where the Data Link Layer handles framing, MAC addressing, and collision avoidance using protocols such as CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance).
