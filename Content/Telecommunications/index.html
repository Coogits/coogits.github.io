    Telecommunications

    Telecommunications refers to the transmission, reception, and exchange of information over long distances using various technologies. This can include voice, data, text, images, and video, transmitted through wired, wireless, optical, or satellite communication systems.

    Models

    OSI Model

    The OSI model is a theoretical framework created by ISO (International Organization for Standardization) to standardize communication across different systems. It is general and flexible, covering many networking concepts, even those not used in TCP/IP.

    1. Physical – Transmission media, cables, signals.
    2. Data Link – MAC addresses, error detection (Ethernet, Wi-Fi).
    3. Network – Logical addressing & routing (IP, ICMP, ARP).
    4. Transport – Reliable or fast delivery (TCP, UDP).
    5. Session – Connection management (SIP, PPTP).
    6. Presentation – Data formatting & encryption (SSL/TLS).
    7. Application – High-level services (HTTP, FTP, SMTP).

    TCP/IP Model

    A simplified version of OSI, used in modern networking
    It is practical, simpler, and directly tied to real protocols (like TCP, IP, and HTTP).

    1. Network Access – Physical and data link layers combined.
    2. Internet – Routing and addressing (IP, ICMP, ARP).
    3. Transport – Communication control (TCP, UDP).
    4. Application – Services for users (HTTP, FTP, SMTP).

    OSI vs. TCP/IP

    However, you can map TCP/IP layers to OSI layers, as TCP/IP roughly follows the same principles but collapses some layers together:

    OSI Model (7 Layers) TCP/IP Model (4 Layers)
    7. Application Application
    6. Presentation (Merged into Application)
    5. Session (Merged into Application)
    4. Transport Transport
    3. Network Internet
    2. Data Link Network Access
    1. Physical (Merged into Network Access)

    Encapsulation/Decapsulation

    • Encapsulation:
      Is the process that the layers are encoded from the higher to the lower level: $$Applicaiton\to Network$$
    • Decapsulation:
      Is the process that the layers are decoded from the lower to the higher level: $$Network\to Application$$

    Fundamental Steps of Data Transmission

    1. Set of Symbols – Define the data or message to be transmitted, represented in a symbolic form (e.g., binary bits, electrical signals, or waveforms).
    2. Encode – Convert the symbols into a transmission-friendly format, such as digital encoding, modulation (AM, FM, QAM), or compression.
    3. Transmit – Send the encoded signal over a communication medium (e.g., copper wire, fiber optics, radio waves).
    4. Propagation Channel – The medium through which the signal travels, which may introduce noise, attenuation, interference, or distortion.
    5. Receive – Capture the transmitted signal at the destination (e.g., an antenna, a network card, or a sensor).
    6. Decode – Process the received signal to extract the original data, involving demodulation, error correction, and signal reconstruction.
    7. Interpret & Use – The final step where the extracted data is understood and utilized by the receiving system or user.
