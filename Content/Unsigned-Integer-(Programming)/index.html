    Unsigned Integer (Programming)

    An unsigned integer is a numeric data type that can represent only non-negative whole numbers (0 and positive values). Unlike signed integers, unsigned integers do not use a sign bit, allowing them to store a larger positive range for the same bit width.

    • Key Points
      • Only stores non-negative values (0 and positive integers).
      • Uses all bits to represent magnitude (no sign bit).
      • Has a larger maximum value compared to its signed counterpart for the same bit width.
      • Overflow occurs when exceeding the maximum value, often wrapping around to 0.
      • Can be mathematically represented as a limited Whole Number Set

    Value Range

    • Unsigned Integer Value Range:
      Since no bits are reserved for the sign (unlike signed integers), the range is twice that of a signed integer: $$2^{n}-1$$ Explicitly: $$\{0,1,\ldots,2^n-1\}$$ where:
      • $n\quad$ is the total number of bits in the integer.

    Common Sizes

    Bit Width Minimum Value Maximum Value
    8-bit 0 255
    16-bit 0 65,535
    32-bit 0 4,294,967,295
    64-bit 0 18,446,744,073,709,551,615

    Special Cases

    Some programming languages do not support unsigned integers explicitly, meaning all integers are signed by default. Examples include:

    • Python:
      Due to it's dynamical-type nature, python don't handle data types directly, including unsigned integers.
    • JavaScript:
      Uses floating-point (Number type) for all numbers, including unsigned integers. Introduces BigInt for true integer arithmetic.

    // JavaScript does not support explicit unsigned integers,

    let my_uint: u32 = 42;

    var myUInt: UInt = 42

val myUByte: UByte = 255u
