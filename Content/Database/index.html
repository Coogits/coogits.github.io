    Database

    A database is an organized collection of data that is stored, managed, and accessed electronically. It allows efficient retrieval, insertion, updating, and deletion of data while ensuring integrity, security, and consistency.

    Concepts

    • Data
      The actual information stored in structured or unstructured formats (entities, attributes, relationship).
    • Schema
      The structure that defines how data is organized (tables, columns, relationships).
    • Queries
      Instructions used to manipulate and retrieve data (e.g., SQL queries).
    • Indexes
      Optimized structures that speed up searches.

    Database Model

    A database model defines how data is structured, stored, and managed in a database. There are different levels of abstraction when designing a database, primarily categorized into Conceptual, Logical, and Physical models.

    • Conceptual Model
      Defines "what" the system represents.
    • Logical Model
      Specifies "how" data is structured.
    • Physical Model
      Details "how" the database is implemented.

    Types of Databases

    Relational Database

    Data is stored in tables with structured relationships. Example: MySQL, PostgreSQL.

    NoSQL Databases

    Handle unstructured or semi-structured data. Example: MongoDB, Redis, XML.

    Hierarchical Databases

    Store data in a tree-like structure with parent-child relationships.

    Object-Oriented Databases

    Store data as objects (used in OOP). Example: db4o.

    Graph Databases

    Designed to store relationships efficiently. Example: Neo4j.

    Time-Series Databases

    Optimized for time-stamped data. Example: InfluxDB.

    Database Management System (DBMS)

    Software that manages the database, e.g.:

    • MySQL
    • PostgreSQL
    • SQLite
    • MongoDB
    • Redis
    • XML
    • db4o
    • Neo4j

    Functions of a Database

    • Data Storage & Organization
      Keeps structured records.
    • Data Retrieval
      Uses queries to fetch information.
    • Data Integrity & Security
      Prevents unauthorized access and maintains accuracy.
    • Concurrency Control
      Ensures multiple users can access data without conflicts.
    • Backup & Recovery
      Protects data from loss or corruption.
