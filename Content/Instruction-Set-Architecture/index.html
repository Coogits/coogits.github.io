    Instruction Set Architecture (ISA)

    Instruction Set Architecture (ISA) is the formal specification of a computerâ€™s capabilities at the software-hardware interface. It defines the set of instructions a processor can execute, the data types it supports, how memory is accessed, and how programs interact with hardware. The ISA serves as the boundary between the software (such as operating systems and applications) and the processor's underlying implementation (microarchitecture).

    Key Components of ISA

    • Instruction Set
      Defines the set of operations a processor can perform, such as:

      • Arithmetic & Logic Operations (e.g., ADD, SUB, AND, OR)
      • Data Movement Instructions (e.g., LOAD, STORE, MOV)
      • Control Flow Instructions (e.g., JUMP, CALL, RETURN, BRANCH)

    • Data Types & Registers
      Specifies the types of data the processor can handle, including:

      • Integer & Floating-Point Numbers
      • Vectors & Specialized Data Types
      • General-Purpose & Special-Purpose Registers

    • Addressing Modes
      Defines how operands are located in memory or registers, such as:

      • Immediate Addressing (value embedded in the instruction)
      • Register Addressing (value stored in a register)
      • Direct & Indirect Addressing (memory-based operand retrieval)

    • Memory Model
      Describes how the processor accesses and manages memory, including:

      • Endianness (Big-Endian vs. Little-Endian)
      • Virtual Memory Support
      • Cache & Memory Hierarchy Considerations

    • Privilege Levels & Exceptions
      Defines system modes (e.g., user mode vs. kernel mode) and how the processor handles exceptions, traps, and interrupts.

    Types of ISAs

    The ISAs are subdivided mostly into RISC vs. CISC Architecture:

    • CISC (Complex Instruction Set Computing):
      Emphasizes powerful, multi-step instructions to reduce code size. (e.g., x86)

    • RISC (Reduced Instruction Set Computing):
      Uses a simpler, fixed-length instruction set optimized for speed and efficiency. (e.g., ARM, RISC-V)
