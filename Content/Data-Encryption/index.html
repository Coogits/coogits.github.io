    Data Encryption

    Data encryption is the process of transforming readable data (plaintext) into an unreadable format (ciphertext) using a mathematical algorithm and a key.
    The purpose of encryption is to protect the confidentiality of the data so that only authorized parties with the correct decryption key can access the original information.

    Purpose and Importance

    • Confidentiality: Prevents unauthorized access to sensitive data.
    • Integrity (with authentication): Some encryption methods also ensure that the data hasnâ€™t been altered.
    • Security in transit and at rest: Encryption is used to secure data both while being transmitted (e.g., over the internet) and while stored (e.g., on a hard drive).

    Basic Concepts

    Term Description
    Plaintext Original, readable data before encryption
    Ciphertext Encrypted data that appears unreadable without a decryption key
    Key A secret value used in encryption and decryption algorithms
    Algorithm A set of rules (mathematical formulas) used to perform encryption/decryption

    Types of Encryption

    1. Symmetric Encryption
      • The same key is used for both encryption and decryption.
      • Fast and efficient for large data volumes.
      • Examples: AES (Advanced Encryption Standard), DES (Data Encryption Standard)
    2. Asymmetric Encryption
      • Uses a public key to encrypt and a private key to decrypt.
      • Often used in secure key exchange and digital signatures.
      • Examples: RSA, ECC (Elliptic Curve Cryptography)
    3. Hybrid Encryption
      • Combines symmetric and asymmetric methods.
      • Common in secure web communication (e.g., HTTPS via TLS).

    Applications of Encryption

    • Secure Web Traffic: HTTPS (uses TLS encryption)
    • Email Protection: PGP (Pretty Good Privacy), S/MIME
    • Data Storage: Disk encryption (BitLocker, FileVault)
    • Messaging Apps: End-to-end encryption (Signal, WhatsApp)

    Encryption in Protocol Layers

    • Presentation Layer: Conceptual location in the OSI model
    • Transport Layer (in practice): TLS/SSL is implemented here
    • Application Layer: Some apps use application-level encryption on top of protocol-based security
