    Integer (Programming)

    An integer is a numeric data type that represents whole numbers, including zero, without any fractional or decimal component. It can be signed (allowing negative values) or unsigned (only non-negative values).

    • Whole Numbers only:
      No fractional or decimal values.
    • Signed vs. Unsigned:
      Can represent negative and positive values or only non-negative values.
      • Signed Integer (Programming)
      • Unsigned Integer (Programming)
    • Binary Representation:
      The Integer Binary Representation follows a positional system, with the most significant bit acting as the sign bit for signed integers.
    • Fixed or Arbitrary Size:
      Some languages use fixed-size integers, while others allow arbitrary precision.
    // Example in C
int myInt = 42; // Signed integer (negative and positive)
unsigned int myUnsignedInt = 42; // Unsigned (positive value only)
long myLong = 123456789L; // Bigger signed integer

    Standard Integer Types

    In most Languages the default int assumed to be unsigned integer unless explicitly stated.

    • Platform-Specific Size
      • C/C++: int is at least 16 bits (per C standard), but most modern compilers implement it as 32-bit.
    • Fixed Size
      • Java, C#: int is always 32-bit.
      • Rust: Uses i32 as the default integer type instead of int.
    • Arbitrary Size
      • Python: Uses arbitrary-precision integers, meaning it expands dynamically as needed.
    • Floating-Point Representation
      • JavaScript: Uses floating-point (Number type) for all numbers, including integers. Introduces BigInt for true integer arithmetic.

    Overflow & Underflow

    • Fixed-size Limits:
      Fixed-size integers (e.g., int, int16, int32) have a limited range, and exceeding this range leads to overflow (wrapping around) or underflow (going below the minimum value).
    • Safeguards:
      Some languages provide safeguards to handle overflow, such as arbitrary-precision integers in Python or checked arithmetic in Rust.

    Performance Considerations

    • Floating Point Comparison:
      Integer operations are generally faster than floating-point operations in most CPUs.
    • Bit Size:
      Choosing smaller integers (e.g., int16 vs. int32) saves memory, but at the cost of a smaller range.
    • Signed vs. Unsigned:
      Since for signed integers one bit is reserved for the sign, unsigned integers can represent double the range value respect it's unsigned counterpart
      for Signed Integers the range value is give by: $$2^{n-1}-1$$ For Unsigned Integers the range value is give by: $$2^{n}-1$$

    Integer Division

    • Truncation:
      In some languages (e.g., C, Java), dividing two integers truncates the result instead of returning a decimal.
      • example: 5 / 2 $\to$ 2 instead of 2.5 in some languages.
      • example with at least one floating-point: (5 / 2.0 $\to$ 2.5).
    // truncation example
int result = 5 / 2; // Output: 2 (integer division, truncates decimal part)
float f = 5 / 2;    // Still 2, because both are int
float f2 = 5 / 2.0; // Output: 2.5 (float division)

    Common Uses

    • Counting and indexing in loops
    • Representing discrete values such as IDs, flags, and enumerations
    • Mathematical calculations where only whole numbers are needed
