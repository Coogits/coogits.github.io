    Character (Programming)

    In programming, a character is a data type that represents a single textual or symbolic unit, such as a letter, digit, punctuation mark, or special symbol. Characters are typically stored using standardized encoding schemes like ASCII or Unicode to ensure compatibility across systems.

    Key Concepts

    • Single Unit:
      Represents a single letter, number, or symbol (e.g., 'A', '7', '$').
    • Encoded Representation:
      Stored as numeric values (e.g., 'A' in ASCII is 65).
    • Immutable:
      In most languages, character values cannot be modified directly.
    • Used in Strings:
      A sequence of characters forms a string.

    Character Encoding

    Since computers store data as binary values, characters need to be encoded numerically to be processed.
    Common Encoding standards include:

    • ASCII (American Standard Code for Information Interchange)
      • Uses 7-bit or 8-bit encoding (values 0-127 or 0-255).
      • Covers basic Latin characters, digits, and symbols.
    • Unicode (Universal Character Set - UCS)
      • Aims to represent all characters in all writing systems.
      • Uses UTF-8, UTF-16, and UTF-32 encodings.
      • UTF-8 is the most common, using variable-length encoding (1–4 bytes).
    • Extended Character Sets
      • ISO-8859: Supports additional Latin characters beyond ASCII.
      • EBCDIC: Used in legacy IBM mainframes.

    Memory and Storage Considerations

    • Fixed vs. Variable Length:
      • ASCII and Latin-1 characters use 1 byte per character.
      • UTF-8 uses 1–4 bytes, depending on the character.
      • UTF-16 uses 2 or 4 bytes, while UTF-32 uses 4 bytes per character.
    • Performance Impact:
      • UTF-8 is widely used for compatibility and efficiency.
      • Fixed-width encodings like UTF-32 can simplify processing but use more memory.
