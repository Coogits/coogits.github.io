    Transport Layer

    The Transport Layer is the fourth layer in the Open Systems Interconnection (OSI) model. Its principal role is to facilitate process-to-process communication, enabling reliable or unreliable data transmission between applications running on different hosts. It acts as an intermediary between the Network Layer (Layer 3), which handles host-to-host communication, and the Session Layer (Layer 5), which manages session control and dialog maintenance.

    At this layer, communication is established end-to-end between the source and destination systems, abstracting the details of the underlying network.

    Primary Responsibilities and Mechanisms:

    1. End-to-End Connection Management

    • Establishes and manages logical connections between applications on different hosts.
    • Supports connection-oriented (e.g., TCP) and connectionless (e.g., UDP) communication.
    • Responsible for connection establishment, maintenance, and termination in reliable protocols.

    2. Segmentation and Reassembly

    • Converts large messages from the Session or Application layers into smaller segments or datagrams suitable for transmission by the Network Layer.
    • At the receiving end, the Transport Layer reassembles these segments into the original message using sequence numbers.

    3. Reliability and Data Integrity

    • Ensures data is delivered accurately and in order, even over unreliable networks.
    • Employs mechanisms such as:
      • Acknowledgment (ACK)
      • Retransmission of lost segments
      • Timeouts
      • Error detection using checksums

    4. Flow Control

    • Prevents the sender from overwhelming the receiver with excessive data.
    • Example: TCP’s Sliding Window Protocol dynamically adjusts the amount of data in transit based on the receiver's buffer availability.

    5. Error Control

    • Includes error detection (via checksums) and error correction (via retransmissions).
    • Ensures the integrity of data through per-segment verification.

    6. Congestion Control

    • Adjusts transmission rate to minimize congestion in the network.
    • Example: TCP congestion control algorithms such as:
      • Slow Start
      • Congestion Avoidance
      • Fast Retransmit/Fast Recovery

    7. Multiplexing and Demultiplexing

    • Enables multiple applications to share the same transport connection.
    • Utilizes port numbers to distinguish between different communication sessions on the same host.
    • Demultiplexing ensures received segments are forwarded to the correct application process.

    Transport Layer Protocols

    Protocol Type Features Use Cases
    TCP (Transmission Control Protocol) Connection-oriented, reliable Sequencing, ACKs, flow & congestion control, full-duplex communication Web, email, file transfer
    UDP (User Datagram Protocol) Connectionless, unreliable Lightweight, minimal overhead, no ACK or retransmissions VoIP, gaming, DNS, streaming
    SCTP (Stream Control Transmission Protocol) Connection-oriented, reliable Supports multi-streaming and multi-homing, message-oriented Telecom, signaling over IP
    DCCP (Datagram Congestion Control Protocol) Connection-oriented, unreliable Adds congestion control to UDP-style transport Streaming media

    Port Numbers:

    Defined by the Internet Assigned Numbers Authority (IANA):

    • Well-Known Ports (0–1023):
      Used by system processes (e.g., HTTP: 80, HTTPS: 443, FTP: 21)
    • Registered Ports (1024–49151):
      Used by user processes and applications
    • Dynamic/Private Ports (49152–65535):
      Used for ephemeral or temporary connections

    Examples of Transport Layer in Practice:

    TCP Example – Web Browsing

    1. Connection Setup: TCP three-way handshake (SYN, SYN-ACK, ACK)
    2. Data Transfer: Reliable delivery using sequence numbers and acknowledgments
    3. Connection Teardown: Graceful termination using FIN/ACK

    UDP Example – Online Gaming

    • Low-latency communication without guarantee of delivery
    • Suitable where timeliness is more important than accuracy

    SCTP Example – Mobile Networks

    • Used in signaling for LTE/5G infrastructure (e.g., between base stations and core network)
    • Provides multi-homing (redundancy across multiple IP paths)

    Relation to the OSI Model:

    • Below: Relies on the Network Layer to route data between hosts.
    • Above: Supports the Session Layer in managing connections between applications.
    • Bridges the gap between host-to-host (Layer 3) and process-to-process (Layer 5) communication.

    Transport Layer Header Structure (Example: TCP)

    • Source Port
    • Destination Port
    • Sequence Number
    • Acknowledgment Number
    • Header Length
    • Flags (SYN, ACK, FIN, etc.)
    • Window Size
    • Checksum
    • Urgent Pointer

    These fields provide the control necessary to ensure ordered, error-checked, and reliable communication.
