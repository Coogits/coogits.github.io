    Transmission Control Protocol (TCP)

    TCP (Transmission Control Protocol) is a stateful, connection-oriented, reliable transport protocol defined in RFC 793 (and extended by later RFCs). It operates at the transport layer of the TCP/IP model and corresponds to Layer 4 in the OSI model. TCP provides process-to-process communication over an unreliable IP layer, offering guaranteed delivery, congestion and flow control, and error checking.

    üîç Technical Definition

    TCP is a byte-stream protocol that allows two endpoints (sockets) to establish a virtual circuit and exchange a continuous stream of data in both directions. TCP handles segmentation, reliability, reordering, retransmission, and error detection, abstracting away the unreliability of the underlying IP layer.

    Core Functionalities

    1. Connection Establishment & Teardown

    • Three-way handshake to establish a full-duplex connection:
      1. SYN ‚Üí (client sends synchronize request)
      2. SYN-ACK ‚Üê (server acknowledges and synchronizes)
      3. ACK ‚Üí (client confirms)
    • Connection teardown involves a four-step FIN/ACK sequence to ensure both sides have completed transmission.

    2. Byte-Stream Abstraction

    • Data is sent as a continuous stream of bytes, not as discrete packets. TCP segments this stream into segments and handles reassembly.

    3. Reliability

    • Ensures in-order, lossless delivery via:
      • Sequence Numbers (for byte tracking)
      • Acknowledgments (ACKs) (explicit confirmation of receipt)
      • Retransmission Timers (if no ACK is received)
      • Duplicate ACKs and Fast Retransmit (for faster loss detection)

    4. Error Detection

    • Each segment includes a checksum calculated over the TCP header, payload, and a pseudo-header including IP information. Corrupt segments are discarded.

    5. Flow Control

    • Implements sliding window protocol using the Window Size field in the header.
    • Prevents the sender from overwhelming the receiver‚Äôs buffer (receiver-advertised window).

    6. Congestion Control

    TCP dynamically adapts to network congestion using algorithms such as:

    • Slow Start
    • Congestion Avoidance
    • Fast Retransmit
    • Fast Recovery This avoids congestion collapse by adjusting the congestion window (cwnd) size based on packet loss and delay signals.

    7. Multiplexing

    • Uses port numbers to allow multiple TCP connections over a single IP address.
    • Provides process-level addressing: (IP address, Port number)

    TCP Segment Header (20‚Äì60 bytes)

    Field Size Description
    Source Port 16 bits Port of the sending process
    Destination Port 16 bits Port of the receiving process
    Sequence Number 32 bits Byte offset of the first byte in this segment
    Acknowledgment Number 32 bits If ACK flag is set, next expected byte
    Data Offset 4 bits Header length (in 32-bit words)
    Reserved 3 bits Reserved for future use
    Flags 9 bits Control bits: SYN, ACK, FIN, RST, PSH, URG, etc.
    Window Size 16 bits Flow control: bytes the receiver can accept
    Checksum 16 bits For error detection
    Urgent Pointer 16 bits Used if URG flag is set
    Options variable E.g., MSS, SACK, Window Scaling
    Padding variable Ensures header is a multiple of 32 bits

    Applications Using TCP

    Application Layer Protocol Uses TCP for...
    HTTP/HTTPS Web traffic
    SMTP/IMAP/POP3 Email services
    FTP File transfers
    SSH/Telnet Remote terminal access
    LDAP Directory services
    RTSP (optionally) Media control signaling

    TCP vs. UDP (Quick Contrast)

    Feature TCP UDP
    Reliability Reliable (ACKs, retransmissions) Unreliable
    Ordering Guarantees ordering No order guarantees
    Overhead Higher (20‚Äì60 byte header) Lower (8 byte header)
    Speed Slower due to checks and controls Faster for time-sensitive data
    Use Cases File transfer, web, email Streaming, VoIP, gaming

    Summary

    TCP provides a virtual circuit abstraction on top of IP, ensuring that applications can communicate safely and reliably, regardless of the complexity and unreliability of the underlying network. It is the backbone of most Internet services and critical for ensuring data integrity, session continuity, and error resilience in network communications.
