    Composite Type

    A Composite Type in programming is a data type that groups multiple values, which can be of the same or different types, into a single entity. Unlike primitive types, which hold a single value, composite types enable complex data structures by combining multiple elements.

    Arguments

    • Array (Programming):
      A fixed-size collection of elements of the same type, stored in contiguous memory locations.

    • Structures:
      A user-defined composite type that groups multiple variables of different types under a single name.

    • Objects:
      An instance of a class, identified by a unique name in object-oriented programming.

    • Lists:
      A dynamic, ordered collection of elements that can grow or shrink in size.

    • Tuple:
      An immutable, ordered collection of elements, often used to store related data.

    • Map:
      A key-value data structure that allows fast lookups based on keys.

    Examples of Composite Types:

    // Array: A collection of elements of the same type.
int numbers[3] = {1, 2, 3};  

    // Structures: A collection of variables of different types grouped together.
struct Point {
    int x;
    int y;
};  

    // Objects (Instances of Classes): Encapsulate data (attributes) and behavior (methods).
class Car {
    string brand;
    int year;
}; 

    # List of integers
numbers = [1, 2, 3, 4]

    # Tuple with two elements
coordinates = (10, 20) 

    # Dictionary (Map)
ages = {"Alice": 25, "Bob": 30}
