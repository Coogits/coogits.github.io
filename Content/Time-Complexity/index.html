    Time Complexity

    Time complexity describes how the runtime of an algorithm grows as the input size increases. Here are the most common time complexities

    Common Types

    O(1) - Constant Time

    • The execution time does not change with input size.
    • Example: Accessing an array element by index
    int arr[5] = {10, 20, 30, 40, 50};
printf("%d", arr[2]);  // Always takes the same time
    • Amortized O(1):
      • Some operations may take more time occasionally, but their average time remains O(1).
      • Example:
        Dynamic array resizing (e.g., std::vector in C++ or Python list).
        • When the array is full, resizing takes O(n)O(n)O(n) (copying elements).
        • However, most insertions take O(1)O(1)O(1), so the average (amortized) time per operation is O(1)O(1)O(1).

    O(log n) - Logarithmic Time

    • The runtime grows slowly as input size increases.
    • Common in divide-and-conquer algorithms and binary search.
    • Example: Binary Search on a sorted array
    int binarySearch(int arr[], int left, int right, int x) {
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == x) return mid;
        else if (arr[mid] < x) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}
    • Use Cases:
      • Binary search
      • Balanced trees (e.g., AVL, Red-Black Tree)

    O(n) - Linear Time

    • The runtime grows proportionally with input size.
    • Example: Iterating through an array
    def find_element(arr, target):
   for element in arr:
       if element == target:
           return True
   return False
    • Use Cases:
      • Looping through an array
      • Finding an element in an unsorted list
      • Copying an array

    O(n log n) - Linearithmic Time

    • Common in efficient sorting algorithms.
    • Example: Merge Sort
    void mergeSort(int arr[], int left, int right) {
    if (left < right) {
        int mid = left + (right - left) / 2;
        mergeSort(arr, left, mid);
        mergeSort(arr, mid + 1, right);
        merge(arr, left, mid, right);  // Merging step (O(n))
    }
}
    • Use Cases:
      • Merge Sort, Quick-Sort (average case)
      • Efficient sorting algorithms

    O(n2) - Quadratic Time

    • Runtime grows rapidly with input size.
    • Example: Bubble Sort (Nested Loops)
    void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(arr[j], arr[j + 1]);
            }
        }
    }
}
    • Use Cases:
      • Brute-force algorithms
      • Nested loops over all pairs of elements

    O(2n) - Exponential Time

    • Runtime doubles with each additional input size.
    • Example: Recursive Fibonacci
    int fibonacci(int n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}
    • Use Cases:
      • Brute-force recursive algorithms (e.g., Towers of Hanoi)
      • Unoptimized recursive problems

    O(n!) - Factorial Time

    • The slowest-growing time complexity.
    • Example: Generating all permutations
    void permute(string str, int l, int r) {
    if (l == r) cout << str << endl;
    else {
        for (int i = l; i <= r; i++) {
            swap(str[l], str[i]);
            permute(str, l + 1, r);
            swap(str[l], str[i]);  // Backtrack
        }
    }
}
    • Use Cases:
      • Brute-force permutations/combinations
      • NP-hard problems (e.g., Traveling Salesman Problem, Sudoku Solver)
