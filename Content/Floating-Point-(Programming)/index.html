    Floating-Point (Programming)

    A floating-point number is a numerical representation that supports real numbers, including fractions and decimals, by storing an approximation using a fixed number of bits. It is commonly used when a wider range and precision are required compared to integer types.

    Key Points

    • Real Number Representation:
      Mathematically represented as a limited Real Number Set
    • Use Scientific Notation:
      Represented internally with Scientific Notation in binary (mantissa x base^exponent).
    • IEE 754 Standard:
      Floating-Point Binary Representation is represented with IEE 754 Standard in most platforms.
    • limited precision:
      Has limited precision, leading to rounding errors.
    • Special Values:
      Supports special values like positive/negative infinity and NaN (Not a Number).

    Floating-Point Arithmetic

    The IEEE 754 standard is the most widely used standard for Floating-Point Binary Representation in computing. It defines the representation and behavior of floating-point numbers in binary and decimal formats across various hardware and software implementations.

    • Floating-Point Number Representation in IEEE 754 A floating-point number is represented as: $$\textbf{Value}=-1^{\text{sign bit}}\cdot\text{mantissa}\times2^{\text{exponent}-\text{bias}}$$

    Common IEEE 754 Floating-Point Formats:

    Format Bit Width Exponent Bits Mantissa Bits Bias
    Half-Precision (binary16) 16 5 10 15
    Single-Precision (binary32) 32 8 23 127
    Double-Precision (binary64) 64 11 52 1023
    Quadruple-Precision (binary128) 128 15 112 16383

    Syntax

    // For both C/C++
float myFloat = 3.14f;    // Single precision (32-bit)
double myDouble = 3.14159; // Double precision (64-bit)
long double myLongDouble = 3.141592653589793L; // Extended precision

    float myFloat = 3.14f;
double myDouble = 3.14159;
decimal myDecimal = 3.141592653589793m; // Higher precision for financial calculations

    float myFloat = 3.14f;    // 'f' suffix required for float
double myDouble = 3.14159; // Default for floating-point literals

    # Default is double-precision floating-point
my_float = 3.14 

    // JavaScript uses IEEE 754 double-precision for all numbers
let myFloat = 3.14; 

    let my_float: f32 = 3.14;  // Single precision
let my_double: f64 = 3.14159; // Double precision (default)

    var myFloat: Float = 3.14  // 32-bit
var myDouble: Double = 3.14159 // 64-bit (default)

    var myFloat float32 = 3.14
var myDouble float64 = 3.14159 // Default floating-point type

    Common Issues

    • Precision errors:
      Not all decimal values can be represented exactly (e.g., 0.1 + 0.2 $\neq$ 0.3 in binary).
    • Overflow/Underflow:
      Numbers too large or small may become infinity or zero.
    • Comparison pitfalls:
      Due to rounding, checking equality (a == b) can be unreliable.
