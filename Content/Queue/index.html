    Queue

    A queue is a linear abstract data type (ADT) that follows the First-In, First-Out (FIFO) principle. This means that elements are inserted at one end (rear) and removed from the other end (front), similar to a real-world queue (e.g., a line at a ticket counter).

    Operators

    • Enqueue():
      Inserts element xxx at the rear of the queue.
    • Dequeue():
      Removes and returns the element from the front of the queue.
    • Front():
      Returns the element at the front without removing it.
    • Rear():
      Returns the element at the rear without removing it.
    • isEmpty():
      Checks if the queue is empty.
    • isFull():
      Checks if the queue is full (for fixed-size implementations).

    Types of Queues

    Linear Queue

    • Basic FIFO structure.
    • Insertions happen at the rear, removals happen at the front.
    • Example: Task scheduling in an operating system.

    Circular Queue

    • The rear wraps around when it reaches the end of the queue (circular indexing).
    • More efficient in memory usage compared to a simple queue.
    • Example: Ring buffer in real-time systems.

    Priority Queue

    • Elements are dequeued based on priority, not the order of insertion.
    • Implemented using a heap or a sorted list.
    • Example: Dijkstraâ€™s algorithm, CPU process scheduling.

    Double Ended Queue

    • Insertions and deletions can happen at both ends (front and rear).
    • Special types: Input-Restricted Deque (insertion at one end only), Output-Restricted Deque (deletion at one end only).
    • Example: Sliding window problems in algorithms.

    Implementation

    Using Arrays

    • Fixed-size implementation.
    • Requires shifting elements after dequeue() (inefficient).

    Using Linked Lists

    • More dynamic (size can grow).
    • Each node contains data and a next pointer.

    Using Stacks

    • A queue can be implemented using two stacks (one for enqueue, one for dequeue).
    • Example: Queue using two stacks in coding interviews.

    Complexity

    Operation Array Implementation Linked List Implementation
    Enqueue $O(1)$ (best case), $O(n)$ (worst case due to shifting) $O(1)$
    Dequeue $O(1)$ $O(1)$
    Front() / Rear() $O(1)$ $O(1)$
    • See Time Complexity for $O(1)$, $O(n)$ meaning

    Code Examples

    C++

    C++ Implementation Using STL Queue

    #include <iostream>
#include <queue>

int main() {
    std::queue<int> q;
    q.push(10); // Enqueue
    q.push(20);
    q.push(30);

    std::cout << "Front: " << q.front() << "\n";
    q.pop(); // Dequeue
    std::cout << "New Front: " << q.front() << "\n";

    return 0;
}

    Python

    Python Implementation Using collections.deque

    from collections import deque

q = deque()
q.append(10)  # Enqueue
q.append(20)
q.append(30)

print("Front:", q[0])
q.popleft()  # Dequeue
print("New Front:", q[0])

    Java

    Java Implementation Using Queue Interface

    import java.util.LinkedList;
import java.util.Queue;

public class Main {
    public static void main(String[] args) {
        Queue<Integer> q = new LinkedList<>();
        q.add(10); // Enqueue
        q.add(20);
        q.add(30);

        System.out.println("Front: " + q.peek());
        q.remove(); // Dequeue
        System.out.println("New Front: " + q.peek());
    }
}
