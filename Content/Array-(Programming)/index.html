    Array (Programming)

    An array is a data structure that stores a fixed-size, ordered collection of elements of the same type in contiguous memory locations. Each element in an array is identified by an index, allowing efficient random access.

    Key Characteristics:

    • Fixed Size:
      The size of an array is typically defined at creation and cannot change dynamically (except in dynamic arrays).
    • Contiguous Memory:
      Elements are stored in adjacent memory locations, enabling fast access.
    • Index-Based Access:
      Elements can be accessed in constant time O(1)[1]O(1) - Constant Time Complexity using their index.
    • Homogeneous Elements:
      All elements in an array must be of the same data type.

    Types of Arrays

    One-Dimensional

    A one-dimensional array (1D array) is the simplest type of array, where elements are stored in a single row (or linearly in memory). Each element is accessed using a single index.

    int numbers[5] = {1, 2, 3, 4, 5};
printf("%d", numbers[2]);  // Output: 3

    Multi-Dimensional

    A multi-dimensional array extends the concept of 1D arrays by adding additional dimensions. The most common type is a two-dimensional array (2D array), often used to represent matrices or tables.

    int matrix[2][3] = {{1, 2, 3}, {4, 5, 6}};
printf("%d", matrix[1][2]);  // Output: 6

    Dynamic Arrays

    A dynamic array is a resizable array that can grow or shrink as needed, unlike static arrays that have a fixed size. Dynamic arrays manage memory allocation automatically. C++ (Vector - Standard Library Dynamic Array):

    #include <vector>
std::vector<int> numbers = {1, 2, 3};
numbers.push_back(4);  // Adds element
numbers.pop_back();    // Removes last element

    Python (List - Built-in Dynamic Array):

    numbers = [1, 2, 3]
numbers.append(4)  # Adds element
numbers.pop()      # Removes last element
print(numbers)  # Output: [1, 2, 3]

    Comparison Table

    Type Memory Allocation Size Flexibility Access Time Use Cases
    One-Dimensional Contiguous Fixed O(1)[1]O(1) - Time Complexity
    Multi-Dimensional Contiguous Fixed O(1) Matrices, tables, images
    Dynamic Array Heap-based Resizable O(1) (amortized) Variable-length data

    Performance

    Lists[3]Linked List are similar data structure, more versatile but require more resources and arrays are similar data structures but differ in terms of performance:

    • Array:
      Pro: Faster, take less memory Cons: Fixed memory, less versatile
    • Lists:
      Pro: Dynamic size, more versatile Cons: Slower, uses more memory

    Operations

    • Access: Retrieve an element using its index O(1)[1]O(1) - Constant Time Complexity.
    • Update: Modify an element at a specific index O(1).
    • Insertion: Adding a new element (costly in static arrays, O(n)[2]O(n) - Linear Time Complexity for shifting).
    • Deletion: Removing an element and shifting remaining elements O(n).
