    Operating System

    An Operating System (OS) is low-level system software that serves as an abstraction layer between hardware and user applications, providing essential functionality for resource management, process execution, and system security. It facilitates interaction with hardware components, optimizes performance, and ensures concurrency, isolation, and security in a multi-user, multi-tasking environment.

    Common OS

    General-Purpose OS

    A General-Purpose Operating System is designed for a wide range of applications, from personal computing to enterprise use. It provides a graphical user interface (GUI), multitasking, and support for various software applications.

    • Windows – A proprietary OS developed by Microsoft, known for its widespread adoption in personal and enterprise computing.
    • Linux – An open-source OS based on the Unix architecture, used in both desktop and server environments.
    • macOS – A Unix-based OS developed by Apple, optimized for Apple hardware with a strong focus on user experience and security.

    Mobile OS

    A Mobile Operating System is optimized for smartphones, tablets, and other portable devices. It prioritizes touch interfaces, power efficiency, and app ecosystems.

    • Android – An open-source OS developed by Google, based on the Linux kernel, widely used across various smartphone brands.
    • iOS – A proprietary OS developed by Apple, used exclusively on iPhones and iPads, known for its tight hardware-software integration.

    Embedded OS

    An Embedded Operating System is designed for specialized devices with dedicated functions, often optimized for low power consumption and real-time performance.

    • FreeRTOS – A lightweight, real-time OS widely used in embedded systems like IoT devices and microcontrollers.
    • RTEMS (Real-Time Executive for Multiprocessor Systems) – A real-time OS designed for aerospace, industrial, and automotive applications.

    Server OS

    A Server Operating System is optimized for handling network requests, managing resources, and running enterprise applications with high reliability and security.

    • Windows Server – A Microsoft OS designed for enterprise environments, supporting Active Directory, virtualization, and cloud integration.
    • Red Hat Enterprise Linux (RHEL) – A commercially supported Linux distribution tailored for enterprise servers and cloud environments.

    OS Core Functions

    • Process Management
      • Handles process creation (fork, exec in Unix-like systems) and termination.
      • Implements scheduling algorithms (FIFO, Round Robin, Priority Scheduling) to optimize CPU utilization.
      • Supports inter-process communication (IPC) through shared memory, message passing, or semaphores.
    • Memory Management
      • Manages physical and virtual memory via paging and segmentation.
      • Uses algorithms like First-Fit, Best-Fit, or Worst-Fit for memory allocation.
      • Employs techniques like demand paging, swapping, and memory protection mechanisms (MMU).
    • File System Management
      • Provides file abstraction through hierarchical structures (FAT32, NTFS, ext4).
      • Controls file permissions, access control lists (ACLs), and journaling mechanisms.
      • Implements I/O buffering, caching, and disk scheduling (C-SCAN, SSTF).
    • Device Management
      • Abstracts hardware devices via drivers and the kernel’s I/O subsystem.
      • Manages interrupts via Interrupt Request Lines (IRQ) and Direct Memory Access (DMA).
      • Implements polling, interrupt-driven I/O, and asynchronous I/O.
    • Security & Protection
      • Enforces user authentication and process isolation.
      • Implements privilege levels (Ring 0 for kernel mode, Ring 3 for user mode).
      • Uses system calls (e.g., chmod, chroot) for access control and sandboxing.
    • Concurrency & Synchronization
      • Provides thread management (user-space vs. kernel-space threads).
      • Implements synchronization primitives like mutexes, spinlocks, semaphores, and condition variables.
      • Prevents deadlocks using algorithms like Banker's Algorithm or priority inheritance.
    • Networking & Communication
      • Implements TCP/IP stack for network communication.
      • Manages socket programming for IPC and distributed computing.
      • Supports network protocols (HTTP, FTP, SSH) via system services.

    OS Types

    • Real-Time OS (RTOS):
      Guarantees deterministic execution (FreeRTOS, VxWorks).
    • Distributed OS:
      Coordinates multiple networked computers (Amoeba, MOSIX).
    • Virtualized OS:
      Runs on hypervisors (VMware ESXi, KVM).

    Kernel

    A kernel is the core component of an operating system that directly interacts with hardware and provides essential services for system processes and applications. It operates in privileged mode (kernel mode) and manages system resources such as CPU, memory, storage, and I/O devices.

    Kernel Functions

    • Process Management
      • Schedules and executes processes using algorithms like Round Robin, Shortest Job Next (SJN), or Priority Scheduling.
      • Manages process creation (fork(), exec()) and termination.
      • Handles context switching between processes and threads.
    • Memory Management
      • Allocates and deallocates memory to processes.
      • Implements paging, segmentation, and virtual memory mechanisms.
      • Uses memory protection techniques to prevent unauthorized access.
    • File System Management
      • Manages file access, creation, and deletion.
      • Implements file system structures like FAT, NTFS, ext4, XFS.
      • Provides system calls (open(), read(), write(), close()).
    • Device Management
      • Interfaces with hardware through device drivers.
      • Uses interrupt handling to process I/O operations efficiently.
      • Supports buffering, caching, and direct memory access (DMA).
    • Inter-process Communication (IPC)
      • Enables processes to communicate using pipes, message queues, shared memory, semaphores.
      • Implements synchronization mechanisms to prevent race conditions.
    • Security & Protection
      • Enforces privilege levels (User Mode vs. Kernel Mode).
      • Implements access control lists (ACLs) and system call restrictions.
      • Provides authentication and encryption for secure operations.

    Kernel Types

    • Monolithic Kernel
      • All essential OS services (memory management, scheduling, file system, drivers) run in kernel space.
      • Fast but large due to direct communication within the kernel.
      • Example: Linux, Unix, Windows NT (early versions).
    • Microkernel
      • Minimal kernel in kernel space, while system services (drivers, file systems) run in user space.
      • More secure and modular, but slower due to message passing overhead.
      • Example: QNX, Minix, L4, macOS XNU (hybrid).
    • Hybrid Kernel
      • Combines monolithic speed with microkernel modularity.
      • Example: Windows NT, macOS (XNU), BeOS.
    • Exokernel
      • Minimalist kernel that exposes hardware to applications.
      • Used for research and performance-critical systems.
      • Example: ExOS, Nemesis.
    • Nanokernel
      • Even smaller than a microkernel, offloading most tasks to external processes.
      • Example: Eros, K42.
