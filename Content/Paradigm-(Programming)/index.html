    Paradigm (Programming)

    • Modular Programming
      • Modularity: Breaking down a program into smaller, reusable units (modules) to make development and maintenance easier.
      • Encapsulation: Each module encapsulates its functionality, limiting interaction with other modules and ensuring that changes to one module donâ€™t affect others.
      • Separation of Concerns: Ensuring that different parts of the program deal with different aspects or concerns (e.g., user interface, data processing).
      • Reusability: Modules are designed to be reused across multiple projects or contexts, improving efficiency and consistency.
      • Inter-module Communication: Defining clear interfaces for how modules interact and share data or control.
    • Procedural Programming
      • Procedure (Function) Calls: Functions are the building blocks that encapsulate reusable logic. These functions are called in a sequence to execute a program.
      • Control Flow: Organizing the flow of execution using conditionals (if-else), loops (for, while), and jumps (goto).
      • Sequential Execution: The logic of the program is executed in a linear sequence, typically from top to bottom.
      • Modularization: Dividing the program into functions that each handle a specific task, improving readability and maintainability.
      • State Management: Handling the flow of program state through variables that retain values between function calls.
    • Object-Oriented Programming (OOP)
      • Class
      • Object
      • Encapsulation: The practice of bundling data and methods that operate on that data, hiding the internal state from outside access.
      • Inheritance
      • Polymorphism
      • Abstraction
