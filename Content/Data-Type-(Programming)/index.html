    Data Type (Programming)

    A Type in programming defines the kind of data a variable, constant, or expression can hold. It determines the possible values, memory allocation, and operations that can be performed on the data.

    It determines:

    • The possible values a variable can take.
    • The memory allocation required.
    • The operations that can be performed on the data.
    • Type safety and compatibility within a program.
    // --- Examples of Data Type in C++ ---
// Primitive Type
int number = 42;                     
// Composite Type (Array)
int numbers[3] = {1, 2, 3};  
// Abstract Type (Generic)
template <typename T> class GenericType {}; // Generic 
// User-Defined Type (Enum) 
enum Color { RED, GREEN, BLUE };

    Common Types

    Primitive Type

    Basic data types like:

    • Integer (Programming)
    • Floating-Point (Programming)
    • Character (Programming)
    • Boolean Type (Programming)

    Composite Type

    Collections of values, such as:

    • Array
    • structures
    • Lists
    • object: An instance of a class, identified by a unique name in object-oriented programming.

    Abstract Types

    Define behavior without specifying implementation, such as:

    • Interfaces
    • Generics
    • Traits

    User-Defined Types

    Custom types created by the programmer using:

    • Classes
    • Structs
    • Enums
    • Type Aliases

    Type Systems

    Programming languages enforce different rules for type safety and conversions.

    • Static Typing:
      Types are checked at compile time (e.g., C, Java, Rust).

    • Dynamic Typing:
      Types are determined at runtime (e.g., Python, JavaScript).

    • Strong Typing:
      Strict rules prevent type mismatches (e.g., Rust, Haskell).

    • Weak Typing:
      Allows implicit conversions between types (e.g., JavaScript, PHP).

    Type Conversion

    • Implicit Conversion (Type Promotion):
      Automatically converting one type to another (e.g., int to float).

    • Explicit Conversion (Casting):
      Manually specifying type conversion ((int) 3.14 in C++).

    Exceptions in Type

    Some dynamically typed languages, like Python, handle types at runtime and may raise exceptions when encountering type mismatches:

    • Type Error:
      Raised when an operation is performed on an inappropriate type.

    • Value-Error:
      Raised when an operation receives an argument of the right type but inappropriate value.

    # --- Examples ---
# TypeError: unsupported operand type(s) for -: 'str' and 'int'
print("Hello" - 5)  
# ValueError: invalid literal for int() with base 10 
int("abc")

    Statically typed languages like C++ or Java may prevent such issues at compile time, but they also provide exception handling mechanisms for runtime errors.
