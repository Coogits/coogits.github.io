    Data Mutability

    Mutability refers to whether a data value can be modified after it has been created.
    It classifies data into two main types:

    • Mutable (Variable)
    • Immutable (Constant)

    Mutable

    Mutable (or Variable) Data can be modified after its creation.

    Mutable Concepts

    • Can be reassigned or modified after initialization.
    • Stored in read-write memory.
    • Used for variables, objects, collections, and data structures requiring updates.

    Mutable Examples

    • Standard variables (e.g., integers, floats, object references in most languages).
    • Lists and dictionaries in Python.
    • Arrays in Java, C++, and JavaScript.
    // --- Example (C++) ---
#include <iostream>
int main() {
	float a = 1.23f; // Declare and initialize a mutable variable
    a = 2.71f;  // Allowed: 'a' is mutable
    std::cout << "Mutable variable a: " << a << std::endl;
    return 0;
}

    Immutable

    Immutable (or constant) data cannot be changed after it is defined.

    Immutable Concepts

    • Once assigned, the value remains unchangeable.
    • Often stored in read-only memory.
    • Useful for defining constants, preventing accidental modifications, and ensuring data integrity.

    Immutable Types

    Immutable data can be subdivided into:

    • Read-Only Variables:
      (Stored in memory but cannot be modified at runtime)
      • Constants (e.g., const in C, final in Java).
      • Strings in some languages (e.g., Python, Java).
    • Compile-Time Constants:
      (Replaced by the compiler or preprocessor, may not exist in memory)
      • Preprocessor macros (e.g., #define PI 3.14159 in C, replaced before compilation).
      • Literal constants (e.g., 42, "hello", directly embedded in machine code).
      • Constant expressions (e.g., constexpr in C++, allowing compile-time evaluation).
    // --- Example (C++) ---
#include <iostream>
#define PI 3.14159  // Preprocessor macro (replaced at compile-time)
int main() {
    const float b = 3.14f; // Immutable constant stored in memory
    constexpr int c = 10;  // Compile-time constant (evaluated at compile-time)
    // b = 2.71f;  // Error: 'b' is read-only
    // PI = 2.71f; // Error: 'PI' is not a variable
    // c = 20;     // Error: 'c' is evaluated at compile-time and is immutable
    std::cout << "Memory-stored constant b: " << b << std::endl;
    std::cout << "Preprocessor macro PI: " << PI << std::endl;
    std::cout << "Compile-time constant c: " << c << std::endl;
    return 0;
}

    In OOP

    In Object-Oriented Programming, mutability extends to objects and class properties:

    • Mutable Objects:
      Objects whose fields can be modified after instantiation.

    • Immutable Objects:
      Objects whose state cannot change after they are created (e.g., String in Java, tuple in Python).

    # --- Immutable Object Example (Python) ---
class ImmutablePoint:
    def __init__(self, x, y):
        self._x = x
        self._y = y
    @property
    def x(self):
        return self._x
    @property
    def y(self):
        return self._y
p = ImmutablePoint(3, 4)
# p.x = 5  # Error: Attribute cannot be modified
print(p.x, p.y)  # Output: 3 4

    Differences

    How to choosing Between Mutable and Immutable:

    Feature Mutable Immutable
    Modification Allowed Not Allowed
    Memory Stored in read-write memory Stored in read-only memory
    Performance Can be modified, may require locking in concurrency Safer in concurrent programming
    Use Cases Variables, collections, configurations Constants, fixed keys, security-sensitive data
