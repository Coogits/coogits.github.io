    Scope

    Scope refers to the visibility and lifetime of an identifier (such as variables, functions, or objects) within a program. It defines where an identifier can be accessed or modified in the code.

    Key Concept

    • Defines Accessibility:
      Determines where an identifier can be referenced.
    • Affects Lifetime:
      Controls how long an identifier exists in memory.
    • Varies by Language:
      Scope rules differ across programming languages.

    Types

    Local Scope

    Local Scope (Block Scope) are identifiers declared within a block ({} or function) are only accessible within that block.
    Exists only inside the function/block

    void example() {
    int x = 10;  // x is local to this function
}
// std::cout << x;  // Error: x is not accessible here

    def example():
    x = 10  # Local variable
print(x)  # Error: x is undefined outside the function

    Global Scope

    Identifiers declared outside all functions are accessible throughout the entire program.
    Accessible anywhere in the program unless shadowed

    int globalVar = 5;  // Global scope
void example() {
    std::cout << globalVar;  // Accessible anywhere
}

    global_var = 5  # Global variable
def example():
    print(global_var)  # Accessible inside the function
example()

    Function Scope

    Identifiers declared inside a function only exist within that function and are destroyed when the function ends.
    Similar to Local Scope but specific to functions.

    function example() {
    let x = 10;
    console.log(x);  // Accessible here
}
console.log(x);  // Error: x is not defined outside
}

    Namespace Scope

    In object-oriented languages, identifiers defined within a class or namespace are accessible within that scope.
    Used in Encapsulation to restrict access to internal members.

    class Example {
    int x;  // x belongs to Example class (class scope)
};

    Static Scope

    Static Scope or Lexical Scope is determined at compile-time, meaning inner blocks inherit variables from outer blocks.
    Nested functions can access outer variables.

    function outer() {
    let x = 10;
    function inner() {
        console.log(x);  // Can access x from outer()
    }
    inner();
}
